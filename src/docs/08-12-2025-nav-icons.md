# Navigation Icon Toggle - December 8, 2025

## Active State Icon Display for Navigation Buttons

### Problem
Navigation buttons only showed text, with no visual indicator of which page the user is currently on. Wanted to show icons when a button is active/selected.

### Solution - Dynamic Icon/Text Toggle

Implemented conditional rendering to show icons when a navigation button is active, while keeping text for inactive buttons.

### Implementation:

#### 1. Updated Footer Component (`footer.tsx`)
```tsx
const { page, setPage } = usePage();

<button 
  className={`leftSide__btn ${page === "aboutMe" ? "leftSide__btn--active" : ""}`}
  onClick={() => setPage("aboutMe")}
>
  {page === "aboutMe" ? (
    <span className="btn-icon">
      <icons.AboutmeLogo />
    </span>
  ) : (
    "About me"
  )}
</button>
```

**Logic:**
- Checks current `page` from context
- Adds `leftSide__btn--active` class when button matches current page
- Shows icon when active, text when inactive
- Applies to all three navigation buttons (About Me, Projects, Skills)

#### 2. Added Navigation Icons (`icons.tsx`)
```tsx
AboutmeLogo: () => (
  <svg>...</svg>  // Person icon
)

ProjectsLogo: () => (
  <svg>...</svg>  // Box with heart icon
)

SkillsLogo: () => (
  <svg>...</svg>  // Lightning bolt icon
)
```

**Icons chosen:**
- **About Me**: Person silhouette (represents user/personal)
- **Projects**: Box with heart (represents project/work with passion)
- **Skills**: Lightning bolt (represents power/abilities)

#### 3. Desktop Styling (`content.css`)
```css
.leftSide__btn {
  min-width: 140px;  /* Maintains button width */
  padding: 0.7rem 2rem;
}

.leftSide__btn--active {
  background-color: var(--hover-color);
  border-color: var(--primary-light-color);
}

.btn-icon {
  width: 24px;
  height: 24px;
}

.btn-icon svg {
  fill: var(--header-primary-light-color);
}
```

**Active state:**
- Background: Cyan hover color
- Border: Brighter cyan
- Icon: White fill, 24px × 24px
- **Size: Fixed at 140px minimum width** (prevents shrinking)

#### 4. Mobile Styling (`content.css` - mobile section)
```css
.leftSide__btn {
  flex: 1;  /* Equal width distribution */
  min-width: unset;  /* Removes desktop min-width */
}

.leftSide__btn--active {
  background-color: var(--secondary-dark-color);
  border-bottom: 3px solid var(--primary-light-color);
}

.btn-icon {
  width: 28px;
  height: 28px;
}
```

**Mobile active state:**
- Background: Secondary dark
- Bottom border: 3px cyan indicator (tab-style)
- Icon: Slightly larger (28px) for touch
- **Width: Each button takes 1/3 of footer** (equal distribution)

### State Flow:

1. User clicks "Projects" button
2. `setPage("projects")` updates context
3. Footer re-renders
4. "Projects" button:
   - Gets `leftSide__btn--active` class
   - Shows ProjectsLogo icon instead of text

5. Other buttons:
   - Lose active class
   - Show text
   - Return to default styling

## Button Size Fix - About Me

### Problem
The "About me" button was shrinking when showing the icon because "About me" (8 characters with space) is longer than "Projects" (8 characters) or "Skills" (6 characters).

### Solution
Increased minimum width from 140px to 160px to accommodate the longest text.

```css
.leftSide__btn {
  min-width: 160px;  /* Fits "About me" comfortably */
}
```

### Result:
✅ All buttons maintain same width when switching to icons  
✅ No layout shift on any button  
✅ Consistent appearance across all navigation items

## Mobile Badge Overflow Fix

### Problem
On mobile project cards, when badges wrapped to two lines, they would overlap the description text above them.

### Root Cause
The card minimum height (110px) wasn't enough to accommodate:
- Project name (1 line)
- Description (2 lines)
- Badges (2 lines when wrapped)

### Solution

**1. Increased Card Height:**
```css
.leftSide__menu_item {
  min-height: 130px;  /* Was 110px */
}
```
- Adds 20px more space
- Accommodates wrapped badges

**2. Reserved Badge Space:**
```css
.leftSide__menu_item_badges {
  min-height: 30px;  /* Reserve space for badges */
}
```
- Ensures badges have minimum space
- Prevents overlap with description

### Card Layout (Mobile):
```
┌─────────────────────────┐
│ Project Name      [1]   │ ← 1 line (flex-shrink: 0)
├─────────────────────────┤
│ Description text that   │ ← 2 lines max
│ might wrap here         │   (flex: 1, line-height: 1.3)
├─────────────────────────┤
│ [Badge] [Badge] [Badge] │ ← Min 30px
│ [Badge]                 │   (flex-shrink: 0, margin-top: auto)
└─────────────────────────┘
   Total: 130px minimum
```

### Result:
✅ **No overlap** - Badges stay below description  
✅ **Proper spacing** - All elements have room  
✅ **Flexible height** - Card grows if needed  
✅ **Clean layout** - Professional appearance
