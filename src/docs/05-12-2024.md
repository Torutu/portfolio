## Decisions & Reasoning

### 1. Single Master Instructions File

**Decision:** Consolidate all documentation into one master file
**Reasoning:** Multiple instruction files create confusion and inconsistency. A single source of truth ensures everyone follows the same standards.

### 2. Date Format DD-MM-YYYY

**Decision:** Changed from `2024-12-05` to `05-12-2024`
**Reasoning:** Matches user's specified format in instructions. Clearer and more consistent with European date format.

### 3. Minimal Function Documentation

**Decision:** Keep only 2 complex function docs (PageProvider, CollapsibleSection)
**Reasoning:** Simple, self-explanatory code doesn't need documentation. Keeps docs focused on truly complex logic only.

### 4. Separate Daily Logs

**Decision:** One file per calendar day for work logs
**Reasoning:** Easier to track progress, see what was done each day, and maintain historical record without mixing content.
